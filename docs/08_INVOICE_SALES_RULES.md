# Invoice and Sales Rules

Sales and invoices are high-risk business operations. AMN RetailOS must prevent duplicate invoices, hidden deletion, partial failures, and untraceable inventory changes.

## Invoice Numbering

- Invoice numbers must never duplicate.
- Invoice numbers must be generated by a backend-controlled sequence.
- Failed sale attempts must not consume invoice numbers unless the final design intentionally records gaps.
- Any allowed gaps must be explainable through logs.

## No Hard Delete

Invoices must not be hard-deleted after creation.

Allowed actions:

- Void
- Cancel
- Return
- Reversal

Sensitive actions require permission, reason, and audit log entry.

## Sale Commit Requirements

A sale must affect these records together:

- Sale
- Sale lines
- Invoice
- Payment records
- Inventory movements
- Customer debt if applicable
- Cash session
- Audit logs

The operation must be transactional. Either all required records are committed or none are committed.

## Idempotency

Payment/save actions must use idempotency keys. A repeated request with the same idempotency key must return the same result and must not create a duplicate sale, invoice, or payment.

## Payment Types

MVP must support:

- Full payment
- Partial payment
- Customer debt
- Mixed payment

Online payments are delayed and must not be included in MVP.

## Returns and Exchanges

Returns must be safe and traceable.

Rules:

- Full and partial returns must be supported.
- Return quantity cannot exceed originally sold quantity unless explicitly handled as a manager-approved exception.
- Inventory effects must be recorded as movements.
- Refund, credit, or debt adjustment must be recorded.
- Audit logs must capture user, reason, and source invoice.

## Held Invoices

Held invoices are allowed as an MVP workflow if implemented safely.

Rules:

- Holding a cart must not reduce inventory unless the backend explicitly implements reservation.
- Held invoices must have clear status.
- Deleting or abandoning held invoices should be permissioned and auditable if sensitive.
