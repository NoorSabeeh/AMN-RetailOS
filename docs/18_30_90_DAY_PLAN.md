# 30/90 Day Plan

## 30-Day Plan

### Week 1: Product Definition, Schema, Workflows, Team Setup

- Freeze MVP scope.
- Confirm Retail/Grocery and basic Wholesale/Grocery as first focus.
- Finalize conceptual schema.
- Finalize core cashier workflow.
- Finalize service contract draft.
- Confirm role-based workflow.
- Prepare Claude UI handoff.

### Week 2: Backend Foundation, Products, Units, Barcode

- Start backend foundation planning.
- Define store setup contracts.
- Define product/category/barcode service behavior.
- Define unit conversion behavior.
- Define database integrity rules.
- Prepare initial backend test scenarios.

### Week 3: POS, Sales, Invoices, Inventory Movement

- Define sale transaction boundary.
- Define invoice lifecycle.
- Define idempotency behavior.
- Define inventory movement rules.
- Define held invoice behavior.
- Define printer failure after sale save behavior.

### Week 4: Users, Audit, Backup, Reports, Demo, Internal Testing

- Define RBAC and approval matrix.
- Define audit log events.
- Define backup/restore workflow.
- Define basic reports.
- Define license/demo status behavior.
- Prepare internal QA checklist.

## 90-Day Plan

### Month 1: Core Data + Sales

- Store setup
- Products, categories, barcodes, units
- Inventory movement foundation
- Sales/POS
- Invoices
- Payments
- Users and roles

### Month 2: Business Safety

- Returns
- Customer debt
- Suppliers and purchases
- Cash sessions
- Reports
- Audit logs
- Backup/restore

### Month 3: Hardening

- License/demo protection
- Installer flow
- Diagnostics
- Update process
- Rollback plan
- Printer Test Center
- Pilot store testing
- Bug fixes
