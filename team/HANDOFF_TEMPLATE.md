# Handoff Template

Copy and paste this template with every Telegram file submission or review request.

```text
Member:
Phase:
Date:
Work type:
Files changed/created:
Summary:

Done:
-

Blocked:
-

Next:
-

Tests/checks done:
-

Screenshots/notes:
-

Needs review from:
-

Do not overwrite:
-

Integration risk:
Low / Medium / High

Recommended merge action:
Review only / Merge after review / Needs changes / Reject for now
```

## Notes

- Use repository-relative paths only, for example `docs/05_BACKEND_ARCHITECTURE.md`.
- Do not include local machine paths.
- Do not include `.env`, `.env.local`, passwords, keys, or private credentials.
- If the submission is code, include the exact build/test command that was run.
- If the submission is documentation only, explain whether links, status, or task scope changed.
